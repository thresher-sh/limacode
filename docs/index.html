// Isolation boundary

The agent gets everything. Your host gives up nothing.

INSIDE THE VM

  • + Full root access
  • + apt-get, npm, pip, cargo
  • + Docker & containers
  • + System tools & daemons
  • + Own kernel & network stack
  • + Your project at ~/workspace/current
VM WALL

BLOCKED FROM HOST

  • × Host filesystem (except mounted dirs)
  • × Host processes & PIDs
  • × SSH keys & credentials
  • × Other VMs & containers
  • × Unrestricted network (optional)
  • × Persistence after exit

Not a container. A real VM — Apple Virtualization on macOS, QEMU on Linux.

// Reference

Commands & options.

limacode Run default agent against current directory
limacode --agent opencode Run a different agent
limacode --env KEY=val Forward environment variables into the VM
limacode --adir name:~/path Mount additional directories
limacode --restrict-dns domains Lock network to specific domains only
limacode list Show running instances
limacode stop [id] Stop and remove an instance
limacode shell Attach to a running VM
limacode build Build the base image locally
limacode config key [value] Get/set persistent defaults
config examples
# Set defaults so you don't have to type them every time $ limacode config agent opencode $ limacode config env ANTHROPIC_API_KEY=sk-ant-... $ limacode config adir shared:~/libs,data:/tmp/data # Then just run $ limacode
